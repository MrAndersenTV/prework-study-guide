HTML

  • Enter your HTML notes here:
    • The head element is a container for meta elements
    • meta elements contain info about the page used by the browser
    • The user does not see head or meta elements
    • "Child" elements are elements nested inside another element
    • The "parent" element is the element that holds the "child" elements
    • For example, a "body" element that holds "h1," "h2," and a "p" element is the parent element. The elements inside the body are the child elements
    • Elements and tags are sometimes used interchangibly
    • However, elements contain the opening and closing tags. If someone refers to a tag as an opening or closing tag, there is a distinction.

CSS

  • Enter your CSS notes here:
    • The following shows an example of a CSS rule:
      • div {color: blue; text-align: justify;}
    • The "div" is an example of the selector. A selector defines the element or attributes to which the rules or declarations apply
    • Each declaration contains a property and a value. The property is the CSS property we want to apply, and then the value defines what it looks like.
    • A CSS selector with multiple declarations is called a declaration block.
    • The CSS box model shows how margins, borders, padding, and content within an HTML element (see image below)
    • Margins determine how much space is on the outside of an element. It is NOT visible.
    • Padding is often confused with margin but isn't the same. Padding adds space INSIDE the element.
    • Adding padding increases the size of the element because the padding is located INSIDE The element.
    • DRY stands for Do Not Repeat Yourself. A coding principle is to encourage developers to reuse or share pieces of code to reduce the number of lines in code. This reduces the size and complexity of the code.
    • Using a percentage rather than a pixel measurement addresses the percentage of the available width of the webpage.
    • This link will give you the hex code for a color.

GIT

  • Enter your GIT notes here:
    • There are a ton of git commands to rememeber
    • Here's the Git Cheat Sheet
    • Here are some important ones to remember
      • git status: checks what branch we are currently on
      • git checkout -b branch-name: creates a new branch AND switches to it
      • git pull branch-name: fetch and merge any commits from the tracking remote branch
      • git push [alias] [branch]: transmit local branch commits to the remote repository
      • Here are the steps to add, commit, and push up the code, then open and merge a pull request:
        1. To save the current feature to GitHub, in Git Bash or the Terminal (for mac) type [git add-A]. the "-A" stands for "Add" and this stages the changes we made.
        2. To commit the code to our repo, type [git commit -m "added CSS to project". the "-m" stands for "merge" and the area in quotes adds notes to the commit.
        3. We can then type [git pull origin main] to ensure our current branch is up to date before opening a pull request.
        4. To push up all of our changes, enter [git push origin feature/add-css]

JavaScript

  • Enter your JavaScript notes here:
    • JavaScript adds interactivity to websites.
    • API stands for Application Programming Interfaces
    • Unlike the CSS code to add CSS to the HTML doc, the code to add JavaScript to your html file goes in the "body" element. We will use the "script" opening and closing tags with the "src" attribute linking the script file to your code.
    • In JavaScript (and CSS), the semicolon marks the end of a statement.
    • The following are core features of the language:
      • Variables: containers that store values. "Let" is used to declare a variable's value. Each variable can have different data types:
        • String: sequence of text. Enclosed in double or single quote marks.
        • Number: Pretty obvious...but numbers don't have quotes around them.
        • Boolean: True/False value. These words do not need quotes either.
        • Array: A structure that lets you store multiple values in a single reference.
        • Object: Can be anything because everything in JavaScript is an object and can be stored in a variable. the above data types can be considered objects.
      • Operators: mathematical symbol that produces a result based on two values or variables. Here are some examples of operators:
        • Addition (+): Adds two numbers together OR combines two strings.
        • Subtraction (-)/Multiplication (*)/Division (/): Do exactly what they do in basic math.
        • Assignment (=): Assigns a value to a variable.
        • Strict equality (===): Tests to see if two values are equal AND of the same data type. It returns a Boolean result.
        • Not (!), Does-not-equal (!==): Returns the logically opposite value of what it tests. It turns a true into a false, etc.
      • Conditionals are code that test if an expression is true or not. The most common conditional structure is the "if...else" statement.
        • The expression after the "if" is the test. If the test is true, it runs the first block of code after the "if" expression. If the test is false, it runs the block of code after the "else" expression.
      • Functions: Allow us to store and package functionality or code that we wish to reuse. This reduces the amount of code we have to repeat.
      • Within functions, there can be a return statement. This statement returns the result variable out of the function so it can be used. This is called varioable scoping.
      • Events: Also called "event handlers." This is where real interactivity comes to a website. The event handler listens for activity in the browser and runs code in response.
      • In order to code an event, you must select an element, then assign the function. The most common event handler function is "addEventListener()".
      • After assigning the function, you pass the name of the event for the code to listen for (for example: "click") and add a functio nto run when the event happens.
        • For example: document.querySelector("html").addEventListener("click", function() {alert("Ouch! Stop poking me!");});
        • Since the "function ()" doesn't have a name, this is called an anonymous function. We can also write anonymous functions using an arrow function that looks like this: () =>
        • The "const" variable keyword is used to lock down what the variable will be when declared. It cannot be reassigned a value later.
