HTML

  • Enter your HTML notes here:
    • The head element is a container for meta elements
    • meta elements contain info about the page used by the browser
    • The user does not see head or meta elements
    • "Child" elements are elements nested inside another element
    • The "parent" element is the element that holds the "child" elements
    • For example, a "body" element that holds "h1," "h2," and a "p" element is the parent element. The elements inside the body are the child elements
    • Elements and tags are sometimes used interchangibly
    • However, elements contain the opening and closing tags. If someone refers to a tag as an opening or closing tag, there is a distinction.

CSS

  • Enter your CSS notes here:
    • The following shows an example of a CSS rule:
      • div {color: blue; text-align: justify;}
    • The "div" is an example of the selector. A selector defines the element or attributes to which the rules or declarations apply
    • Each declaration contains a property and a value. The property is the CSS property we want to apply, and then the value defines what it looks like.
    • A CSS selector with multiple declarations is called a declaration block.
    • The CSS box model shows how margins, borders, padding, and content within an HTML element (see image below)
    • Margins determine how much space is on the outside of an element. It is NOT visible.
    • Padding is often confused with margin but isn't the same. Padding adds space INSIDE the element.
    • Adding padding increases the size of the element because the padding is located INSIDE The element.
    • DRY stands for Do Not Repeat Yourself. A coding principle is to encourage developers to reuse or share pieces of code to reduce the number of lines in code. This reduces the size and complexity of the code.
    • Using a percentage rather than a pixel measurement addresses the percentage of the available width of the webpage.

GIT

  • Enter your GIT notes here:
    • There are a ton of git commands to rememeber
    • Here's the Git Cheat Sheet
    • Here are some important ones to remember
      • git status: checks what branch we are currently on
      • git checkout -b branch-name: creates a new branch AND switches to it
      • git pull branch-name: fetch and merge any commits from the tracking remote branch
      • git push [alias] [branch]: transmit local branch commits to the remote repository
      • Here are the steps to add, commit, and push up the code, then open and merge a pull request:
        1. To save the current feature to GitHub, in Git Bash or the Terminal (for mac) type [git add-A]. the "-A" stands for "Add" and this stages the changes we made.
        2. To commit the code to our repo, type [git commit -m "added CSS to project". the "-m" stands for "merge" and the area in quotes adds notes to the commit.
        3. We can then type [git pull origin main] to ensure our current branch is up to date before opening a pull request.
        4. To push up all of our changes, enter [git push origin feature/add-css]
